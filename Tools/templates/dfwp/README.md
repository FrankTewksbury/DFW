# DFWP

This project follows the DFW methodology. See `.dfw/project.json` for project identity and `.dfw/constitution.json` for conventions.

## Directory Structure

| Directory | Purpose |
|-----------|---------|
| `.dfw/` | Project identity, constitution, personal config |
| `docs/` | Specs, architecture, ADRs, guides |
| `plans/` | TODOs, roadmaps, wishlists, sprint plans |
| `prompts/` | System prompts, handoffs, reusable templates |
| `context/` | Active context, decisions log, retrospectives |
| `research/` | Research artifacts, analysis, findings |
| `scripts/` | Automation scripts |

## Getting Started

1. Read `docs/DFW-CONSTITUTION.md` for the methodology principles
2. Read `docs/DFW-OPERATING-MANUAL.md` for the full methodology
3. Check `context/_ACTIVE_CONTEXT.md` for current project state
4. Check `plans/_TODO.md` for actionable tasks
